# Remove Tracked Files from Git (Keep Local Copies)

These commands remove files from **Git tracking only**. Your local files on disk are **not deleted**.

Run all commands from the repository root:

```powershell
cd "D:\Stream speech"
```

---

## Step 1 — Ensure .gitignore exists

The root `.gitignore` must exist before running `git add` again.

```powershell
# Verify
Get-Content .gitignore | Select-Object -First 5
```

---

## Step 2 — Remove large / generated paths from Git index

```powershell
# Python virtual environment (~3 GB, 53k files)
git rm -r --cached backend/venv/

# Model weights & caches (~160 MB)
git rm -r --cached backend/pretrained_models/

# Python bytecode
git rm -r --cached backend/__pycache__/

# Generated / temp audio
git rm -r --cached backend/temp_audio/
git rm -r --cached backend/static/audio/

# Runtime database
git rm -r --cached backend/data/

# Frontend build output
git rm -r --cached streamspeech/dist/

# Metrics output (regenerated by script)
git rm -r --cached backend/metrics_output/

# Unused lockfile
git rm --cached pnpm-lock.yaml 2>$null

# TypeScript build info (if tracked)
git rm --cached streamspeech/tsconfig.tsbuildinfo 2>$null
```

---

## Step 3 — Verify nothing large remains tracked

```powershell
git ls-files | ForEach-Object {
  if (Test-Path $_) {
    $s = (Get-Item $_).Length
    if ($s -gt 50MB) { "$([math]::Round($s/1MB,2)) MB  $_" }
  }
}
```

Expected output: **empty** (no files over 50 MB).

```powershell
git ls-files | Measure-Object -Line
```

Expected: **~150–200 files** (not 53,000+).

---

## Step 4 — Stage .gitignore and commit

```powershell
git add .gitignore
git add GIT_CLEANUP_REPORT.md REMOVE_TRACKED_FILES.md FINAL_GITHUB_CHECKLIST.md PUSH_TO_GITHUB.md
git status
```

Review `git status` — you should see thousands of **deletions** from the index and the new ignore rules staged.

```powershell
git commit -m "Remove venv, models, and generated files from tracking; add root .gitignore"
```

---

## Step 5 — Push

```powershell
git push -u origin main
```

If push still fails due to **history containing large blobs**, use **Option A** in `PUSH_TO_GITHUB.md` (fresh history).

---

## What stays on disk (not deleted)

| Path | Purpose |
|---|---|
| `backend/venv/` | Your working Python environment |
| `backend/pretrained_models/` | Downloaded model cache |
| `backend/static/audio/` | Generated MP3 files |
| `backend/temp_audio/` | Temp upload files |
| `streamspeech/node_modules/` | npm packages |
| `streamspeech/dist/` | Production build |

These are simply no longer tracked by Git.
